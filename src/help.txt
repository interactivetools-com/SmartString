SmartString: XSS-Safe Strings with Automatic HTML Encoding and Chainable Utility Methods
==========================================================================================
SmartString automatically HTML-encodes output in string contexts for XSS protection and
provides chainable utility methods for encoding, formatting, and text manipulation.

Basics
-------
$str = SmartString::new("It's easy!<hr>");    // wrap any value (string, int, float, bool, null)
echo $str;                                    // "It&apos;s easy!&lt;hr&gt;" - HTML-encodes automatically in string contexts
echo $str->value();                           // "It's easy!<hr>" - the original value (the escape hatch)
echo $str->trim()->maxChars(60)->or('None');  // methods chain left to right
print_r($str);                                // show object value in a readable debug format
SmartString::help();                          // displays this help text (works on values too: $str->help())

Type Conversion
----------------
Return the value as a plain PHP type and end the chain.

->value()                  Returns the original, unencoded value in its original type
->string()                 Returns value as string (original value, not HTML-encoded)
->int()                    Returns value as integer
->float()                  Returns value as float
->bool()                   Returns value as boolean
SmartString::getRawValue() Returns original value from Smart* objects, other types pass through unchanged

Encoding
---------
Return the encoded value as a plain string and end the chain. Missing values (null or "") return "".

->htmlEncode()             Returns HTML-encoded string (same as automatic encoding, for non-string contexts)
->urlEncode()              Returns URL-encoded string, example: "?user={$user->name->urlEncode()}"
->jsonEncode()             Returns JSON-encoded string, example: "let user={$user->name->jsonEncode()}" (malformed UTF-8 becomes �)
->nl2br()                  HTML-encodes special chars, then converts newlines to <br> tags
->appendHtml('<br>')       HTML-encodes the value, then appends your trusted markup as-is
->wrapHtml('<h2>','</h2>') HTML-encodes the value, then wraps it in your trusted markup as-is
->rawHtml()                Alias for ->value(), reads clearly when outputting trusted HTML on purpose

String Manipulation
--------------------
Return a new SmartString, so you can keep chaining. Missing values (null or "") come through
unchanged, so a later ->or() fallback still works.

->append(' items')                      Adds text to the end of the value (zero counts as present)
->prepend('Tel: ')                      Adds text to the beginning of the value
->wrap('(', ')')                        Wraps the value; pass "" for a side you don't want
->textOnly()                            Removes HTML tags, decodes entities, and trims whitespace
->trim(...)                             Trims whitespace, supports the same parameters as PHP trim()
->maxWords($max, $ellipsis = '...')     Limits value to $max words, adds $ellipsis if cut
->maxChars($max, $ellipsis = '...')     Limits value to $max characters, adds $ellipsis if cut
->pregReplace($pattern, $replacement)   Replaces text matching a regex pattern

Dates & Numbers
----------------
Return a new SmartString, so you can keep chaining. Missing or invalid input returns null -
add ->or() after to show a fallback.

->dateFormat(...)                       Formats as a date or datetime, e.g. dateFormat('Y-m-d H:i:s'); default: SmartString::$dateFormat
->numberFormat($decimals)               Formats as a number with thousands separators, e.g. numberFormat(2)
->percent($decimals, $ifZero)           Converts a decimal to a percentage, e.g. 0.5 becomes 50% ($ifZero shown for zero)
->percentOf($total, $decimals)          What percentage the value is of $total, e.g. 24 of 100 is 24%
->add($value)                           Adds $value
->subtract($value)                      Subtracts $value
->multiply($value)                      Multiplies by $value
->divide($divisor)                      Divides by $divisor

Conditional Replacement
------------------------
Return a new SmartString. Each swaps in a replacement value when its condition matches.

->or('replacement')                     Replaces missing values (null or ""); zero counts as present
->ifNull('replacement')                 Replaces null
->ifZero('replacement')                 Replaces zero (0, 0.0, "0", or "0.0")
->ifTrue($condition, $newValue)         Replaces the value when your condition is truthy
->ifEquals($match, $newValue)           Replaces the value when it loosely equals $match (==, "5" matches 5)
->set($newValue)                        Replaces the value unconditionally, useful for match() results or calculations

Require a Value
----------------
Use these for values that must exist, like a record ID from the URL. If the value is missing
(null or "") they stop the page; otherwise they do nothing and the chain continues.
Zero counts as present. Text is HTML-encoded automatically before being output or thrown.

->orDie('text')            Outputs the message and exits
->or404('text')            Outputs a 404 header and the message, then exits
->orThrow('text')          Throws an Exception with the message
->orRedirect('url')        Redirects to the URL and exits

Value Checks
-------------
Return a plain true or false, typically used in if statements.

->isEmpty()                Returns true if the value is empty ("", null, false, 0, "0"), uses PHP empty()
->isNotEmpty()             Returns true if the value has content - the exact opposite of isEmpty()
->isMissing()              Returns true if the value is missing (null or ""); zero counts as present
                           (zero: isEmpty() true, isMissing() false - use isMissing() when a real 0 must count as present)
->isNull()                 Returns true if the value is null

Custom Functions
-----------------
->map($func, ...$args)     Calls your function with the original value and returns the result
                           as a new SmartString - runs even when the value is null

Setting defaults (at the top of your script or in an init file)
----------------------------------------------------------------
SmartString::$numberFormatDecimal   = '.';             // Default decimal separator
SmartString::$numberFormatThousands = ',';             // Default thousands separator
SmartString::$dateFormat            = 'Y-m-d';         // Default dateFormat() format

Working with arrays? SmartArray is the companion library that handles arrays (like database
rows) as collections of SmartStrings: $user = SmartArrayHtml::new($record);
It has its own help: SmartArrayHtml::new([])->help()

Full documentation: https://github.com/interactivetools-com/SmartString
